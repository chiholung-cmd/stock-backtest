# AlphaTest — 美股策略回測平台

A modern stock strategy backtesting platform built with React 19, TypeScript, tRPC, and Drizzle ORM. Fully self-contained — no third-party OAuth required.

## Features

- **4 Built-in Strategies**: MA Crossover, RSI Overbought/Oversold, MACD, Bollinger Bands
- **Custom Parameters**: Adjust strategy parameters via sliders
- **Yahoo Finance Data**: Free real-time US stock data via `yahoo-finance2`
- **TradingView Widget**: Embedded K-line chart
- **Performance Metrics**: Annualized return, max drawdown, Sharpe ratio, win rate, total trades
- **Equity Curve**: Interactive area chart with Recharts
- **History & Compare**: Save results and compare multiple backtests side-by-side
- **Local Auth**: Email/password registration and login — no OAuth dependency

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | React 19, Tailwind CSS 4, Recharts, TradingView Widget |
| Backend | Node.js, Express, tRPC 11 |
| Auth | Local JWT (bcryptjs + jose) — email/password |
| Database | MySQL / TiDB (Drizzle ORM) |
| Data Source | Yahoo Finance (yahoo-finance2 v3) |

## Deploy on Render

### Prerequisites

A MySQL-compatible database. Free options:
- [PlanetScale](https://planetscale.com) — free tier available
- [TiDB Cloud](https://tidbcloud.com) — free tier available
- [Railway MySQL](https://railway.app) — free tier available

### Steps

1. Go to [render.com](https://render.com) → **New → Web Service**
2. Connect your GitHub account and select the `stock-backtest` repository
3. Render will auto-detect `render.yaml` — confirm the settings
4. Set the following **Environment Variables** in the Render dashboard:

| Variable | Required | Description |
|---|---|---|
| `DATABASE_URL` | ✅ Yes | MySQL connection string: `mysql://user:pass@host:3306/dbname` |
| `JWT_SECRET` | ✅ Yes | Auto-generated by render.yaml, or set your own random string |
| `NODE_ENV` | ✅ Yes | Set to `production` (auto-set by render.yaml) |

5. Click **Deploy** — Render will run `pnpm install && pnpm run build`, then start with `pnpm run start`

### Database Migration

After first deploy, connect to your database and run the SQL files in `drizzle/` directory in order:
- `drizzle/0000_*.sql` — creates users table
- `drizzle/0001_*.sql` — creates backtest_results table
- `drizzle/0002_*.sql` — updates users table for local auth

## Local Development

```bash
# Install dependencies
pnpm install

# Set environment variables
export DATABASE_URL="mysql://user:pass@localhost:3306/stockbacktest"
export JWT_SECRET="your-random-secret"

# Start dev server
pnpm run dev
```

## Auth Flow

- `POST /api/auth/register` — create account with email + password
- `POST /api/auth/login` — sign in, receive session cookie
- `POST /api/auth/logout` — clear session cookie
- `GET /api/auth/me` — get current user from cookie
